Questions and Answers

Q1: What is the purpose of the `MiniGame` interface?

A1: The `MiniGame` interface defines a contract for any mini-game, requiring a `play()` method that returns an integer. It allows various game types to be structured in the same way, even though their internal logic may differ.

Q2: What type of method is declared in the `MiniGame` interface?

A2: The `MiniGame` interface declares an abstract method `play()` that returns an integer. Abstract methods have no body in interfaces and must be implemented by any class that implements the interface.

Q3: How does the `MiniGame` interface relate to the `NumberGuessingGame` class?

A3: The `NumberGuessingGame` class implements the `MiniGame` interface, meaning it provides its own implementation of the `play()` method, defining the logic for the number guessing game.

Q4: Why would you use an interface like `MiniGame` in an application?

A4: Using an interface like `MiniGame` allows you to define a common structure for different mini-games. It enables polymorphism, where different game types can be treated uniformly, despite having different implementations.

Q5: Can you instantiate the `MiniGame` interface directly in Java?

A5: No, you cannot instantiate an interface in Java. You can only instantiate a class that implements the interface and provides an implementation for its methods.

A6: Yes, multiple classes can implement the `MiniGame` interface, each providing a different implementation of the `play()` method. For example, you could have `NumberGuessingGame`, `TicTacToe`, and `QuizGame` all implementing the same interface.

A7: If a class implements the `MiniGame` interface but does not provide an implementation for the `play()` method, it will result in a compile-time error. A class that implements an interface must provide concrete implementations for all abstract methods in that interface.

A8: The `int` return type allows the `play()` method to return a numeric result, such as a score or outcome of the game. This provides a flexible way to represent different types of game results, which can then be used by the calling code.

A10: The `play()` method is abstract because the `MiniGame` interface is designed to define a common behavior for all mini-games, but the specific details of how the game is played will vary between different classes. The method must be implemented in the classes that implement the interface.
